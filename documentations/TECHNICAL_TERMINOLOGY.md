## Glossary (Detailed Definitions)

### ðŸ”´ HIGHLY TECHNICAL TERMS

#### **Memoization**
**Category**: Advanced Computer Science
**Definition**: A performance optimization technique that caches the results of expensive function calls and returns the cached result when the same inputs occur again. In React, used via `useMemo` and `useCallback` hooks.
**AdminEstate Usage**: Analytics calculations are memoized to prevent recalculating on every render.

#### **IndexedDB**
**Category**: Web Technologies
**Definition**: A low-level browser API for client-side storage of significant amounts of structured data, including files/blobs. A NoSQL database built into modern browsers.
**AdminEstate Usage**: Primary offline storage for properties, tenants, work orders, and transactions.

#### **OCR (Optical Character Recognition)**
**Category**: Advanced Algorithms
**Definition**: Technology that recognizes text within images and converts it to machine-readable text data using computer vision and machine learning.
**AdminEstate Usage**: pytesseract (Python) and tesseract.js (JavaScript) for extracting text from scanned documents.

#### **Risk Scoring Algorithm**
**Category**: Custom Algorithm
**Definition**: Automated tenant evaluation system calculating a 0-100 score based on income verification (3x rent rule), employment stability, rental history, and reference completeness.
**AdminEstate Usage**: ApplicationDetails.jsx calculates risk scores to help landlords make data-driven decisions.

#### **Props Drilling**
**Category**: React Anti-Pattern
**Definition**: Passing data through multiple component layers to reach deeply nested children. Can make code harder to maintain.
**AdminEstate Usage**: Main app uses centralized state with controlled props drilling (acceptable for app size per architecture docs).

#### **Error Boundary**
**Category**: React Pattern
**Definition**: React component that catches JavaScript errors anywhere in child component tree, logs errors, and displays fallback UI instead of crashing.
**AdminEstate Usage**: ErrorBoundary.js wraps app to gracefully handle runtime errors.

#### **Async/await Pattern**
**Category**: JavaScript Pattern
**Definition**: Modern JavaScript syntax for handling asynchronous operations, making async code look synchronous and easier to read.
**AdminEstate Usage**: All CRUD operations in usePropertyData.js use async/await for data persistence.

#### **useMemo Hook**
**Category**: React Hook
**Definition**: React hook that returns a memoized value, only recomputing when dependencies change. Prevents expensive calculations on every render.
**AdminEstate Usage**: useAnalytics.js memoizes financial stats and expense categories.

#### **useCallback Hook**
**Category**: React Hook
**Definition**: React hook that returns a memoized callback function, preventing child component re-renders when function reference doesn't change.
**AdminEstate Usage**: Event handlers like handleSearchChange and handleSubmit are wrapped in useCallback.

#### **REST API**
**Category**: Web Architecture
**Definition**: Representational State Transfer - architectural style for networked applications using HTTP methods (GET, POST, PUT, DELETE) on resources.
**AdminEstate Usage**: Flask backend provides REST endpoints for properties, tenants, work orders, and transactions.

#### **CORS (Cross-Origin Resource Sharing)**
**Category**: Web Security
**Definition**: Browser security feature that controls how web pages in one domain can access resources from another domain.
**AdminEstate Usage**: Flask backend configured with CORS headers to allow frontend API access.

#### **CRUD Operations**
**Category**: Data Pattern
**Definition**: Create, Read, Update, Delete - the four basic operations for persistent storage.
**AdminEstate Usage**: usePropertyData.js implements full CRUD for all entities (properties, tenants, work orders, transactions, documents, applications).

#### **Offline-first Architecture**
**Category**: Architecture Pattern
**Definition**: App design where data is stored locally first, app works without internet, and syncs to cloud when connection available.
**AdminEstate Usage**: IndexedDB stores all data locally, localStorage for preferences, with optional backend sync.

#### **Context API**
**Category**: React Pattern
**Definition**: React's built-in solution for passing data through component tree without props drilling. Creates global state accessible to any component.
**AdminEstate Usage**: Not currently used (documented as future enhancement if props drilling becomes problematic).

#### **webpack**
**Category**: Build Tool
**Definition**: Module bundler that takes JavaScript modules with dependencies and generates static assets representing those modules.
**AdminEstate Usage**: Part of Create React App build pipeline for bundling, minification, and optimization.

#### **pandas**
**Category**: Python Library
**Definition**: Python data manipulation library providing DataFrames (2D tables) with powerful operations for cleaning, transforming, and analyzing data.
**AdminEstate Usage**: Backend analytics for advanced financial reports and data insights.

#### **pytesseract**
**Category**: Python Library
**Definition**: Python wrapper for Tesseract OCR engine, enabling text extraction from images (scanned documents, screenshots).
**AdminEstate Usage**: document_processor.py uses pytesseract for image-based document processing.

#### **mammoth.js**
**Category**: JavaScript Library
**Definition**: JavaScript library that converts DOCX (Word) documents to HTML with clean, simple markup.
**AdminEstate Usage**: Client-side DOCX processing for document preview and text extraction.

#### **Tailwind CSS**
**Category**: CSS Framework
**Definition**: Utility-first CSS framework providing low-level utility classes for building custom designs without writing CSS.
**AdminEstate Usage**: All component styling uses Tailwind classes (e.g., `flex`, `bg-blue-600`, `rounded-lg`).

#### **FastAPI**
**Category**: Python Framework
**Definition**: Modern, fast Python web framework for building APIs with automatic interactive documentation and type validation.
**AdminEstate Usage**: Alternative backend option (configured in requirements.txt alongside Flask).
